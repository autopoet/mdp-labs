## 实验目标

VQ，Vector Quantization，中文叫矢量量化。

它的目标是：用有限数量的代表向量来近似大量原始样本。

在本实验中，每个样本是一个 32 维颜色直方图特征。VQ 会生成若干个代表性的 32 维向量，这些代表向量叫做码字，所有码字组成码书。

每个原始样本都可以用离它最近的码字来近似表示。

简单说就是：

```text
原始大量样本 -> 找出若干代表样本 -> 每个样本用最近代表来表示
```

## 基本概念

### 1. 码字

码字就是一个代表向量。

在本实验中，一个码字也是 32 维的，因为原始颜色直方图特征是 32 维。

### 2. 码书

所有码字放在一起，就是码书。

如果码字数量是 `K`，那么码书中就有 `K` 个 32 维向量。

### 3. 码书索引

每个样本量化后，不再直接保存完整的 32 维向量，而是保存它对应的最近码字编号。

例如：

```text
sample_id = 1, codeword_index = 52
```

表示第 1 个样本可以用码书中编号为 52 的码字来近似表示。

### 4. 失真度

量化一定会带来误差，因为样本被最近码字代替后，通常不可能完全一样。

失真度就是用来衡量这种误差的指标。

本实验中使用平均均方误差：

```text
distortion = 所有样本到最近码字的平方距离和 / (样本数 * 特征维数)
```

失真度越小，说明码书对原始数据的近似效果越好。

## 原理讲解

### 1. 为什么要 Min-Max 归一化

不同特征维度的取值范围可能不同。

如果某一维数值范围特别大，那么计算欧氏距离时，这一维会对距离产生更大影响，导致其他维度被忽略。

因此在 VQ 之前，需要把每一维都缩放到 `[0, 1]`：

```text
x' = (x - min) / (max - min)
```

这样所有维度处于相近范围，计算距离时更加公平。

### 2. 为什么要用 K-means 生成码书

VQ 的关键是找到一组有代表性的码字。

K-means 的思想正好适合这个任务：

1. 先给出 `K` 个初始码字；
2. 每个样本找离自己最近的码字；
3. 每个码字移动到属于它的样本均值位置；
4. 重复这个过程，直到码字基本稳定。

最终得到的 `K` 个中心点，就是 VQ 的码书。

### 3. 动态码字数量

题目提示尽量使用动态码字数量。

本代码没有固定写死一个常数，而是根据：

- 样本数量；
- 特征维数；

动态计算码字数量。

当前数据集运行得到：

```text
K = 132
```

这样比直接写死一个固定码字数量更符合实验要求。

### 4. 量化过程

有了码书以后，每个样本都计算它到所有码字的距离，选择最近的那个码字作为近似结果。

距离公式为：

```text
dist(x, c) = sum((x_j - c_j)^2)
```

其中：

- `x` 是样本向量；
- `c` 是码字向量；
- `j` 表示第 j 个特征维度。

最终每个样本都会得到一个码书索引。

## 本实验流程

1. 读取 `ColorHistogram.asc`。
2. 跳过第一列编号，只取后 32 列特征。
3. 对所有特征做 Min-Max 归一化。
4. 根据样本数量和特征维数动态计算码字数量 `K`。
5. 初始化码书。
6. 使用 K-means 迭代优化码书：
   - 分配样本到最近码字；
   - 根据所属样本均值更新码字；
   - 重复迭代。
7. 用最终码书重新量化所有样本。
8. 输出码书、码字数量、失真度和每个样本的码书索引。

## 代码块讲解

验收时如果需要讲代码，不需要逐行解释，按下面几个代码块说明即可。

### 1. 路径和输出目录

代码开头定义：

- 数据集路径；
- 输出目录；
- 当前脚本目录。

作用是保证脚本能读取 `ColorHistogram.asc`，并把结果保存到 `output_py`。

### 2. `load_samples()`

这个函数负责读取数据。

重点：

- 第一列是样本编号；
- 后 32 列是特征；
- 函数返回两部分：`ids` 和 `x`。

其中 `ids` 用于最后输出样本索引，`x` 用于 VQ 计算。

### 3. `minmax_normalize()`

这个函数负责归一化。

它对每一维分别计算最小值和最大值，然后把该维所有数据缩放到 `[0, 1]`。

归一化参数 `min` 和 `max` 会保存到 `vq_summary.json`，方便说明数据是如何缩放的。

### 4. `dynamic_k()`

这个函数负责动态确定码字数量。

验收时可以这样说：

```text
我没有固定写死码字数，而是根据样本数和特征维数动态计算 K。
当前数据集得到 K=132。
```

### 5. `init_codebook()`

这个函数负责初始化码书。

它的作用是给 K-means 一个初始码书。

代码先选择离整体均值最近的样本作为第一个码字，然后不断选择离当前码书最远的样本作为新码字。这样初始码字分布更分散，结果也更稳定。

### 6. `assign_samples()`

这个函数负责把每个样本分配给最近的码字。

它完成两件事：

1. 计算样本到各个码字的距离；
2. 找到距离最小的码字编号。

输出的 `index` 就是每个样本对应的码书索引。

### 7. `kmeans()`

这个函数是 VQ 的核心迭代过程。

每轮迭代做两步：

1. 分配：每个样本找最近码字；
2. 更新：每个码字变成属于它的所有样本的均值。

每轮都会计算失真度，失真度逐渐下降，说明码书越来越能代表原始数据。

### 8. 结果输出

代码最后输出三个文件：

- `vq_codebook.csv`：保存每个码字的 32 维数值；
- `vq_indices.csv`：保存每个样本对应的码字编号；
- `vq_summary.json`：保存码字数量、失真度、归一化参数。
